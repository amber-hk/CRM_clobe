# 액션 기반 필요 사항

> CRM 프론트에서 **버튼은 있는데 뒤에서 안 일어나는 것들** 목록.
> DB 스키마·API 스펙은 `data-requirements.md` 참고.

---

## 1. 발송

### 이메일 실발송

| CRM에서 되는 것 | 안 되는 것 |
|----------------|-----------|
| 제목/본문 작성, 변수 치환 미리보기 | 실제 NHN 이메일 API 호출 |
| CSV 파싱 + 수신자 확인 | 서버에서 수신자 순회 발송 |
| 테스트 발송 버튼 | 이메일 테스트 발송 (현재 stub) |

### 알림톡 실발송

| CRM에서 되는 것 | 안 되는 것 |
|----------------|-----------|
| 템플릿 선택 + 변수 입력 + 미리보기 | 대량 발송 (1000건 이상) |
| 테스트 발송 (단건 실호출) | CSV 기반 배치 발송 |

### 대량 발송 처리

- 5000건 이상이면 동기 API로 불가 → 비동기 큐 필요
- NHN 알림톡은 1회 최대 1000건 → 청크 분할
- NHN 일시 오류(429/5xx) → 재시도 로직
- 발송 중 진행률 표시 → 상태 폴링

## 2. 결과 수집

### 이메일 오픈 추적

현재: 오픈 감지 수단 없음. 오픈율 = null.

**방식 선택 필요**:
- A) NHN 수신확인 웹훅 → 콜백 URL 등록
- B) 자체 트래킹 픽셀 → 이메일 본문에 1x1 이미지 삽입

### 클릭 추적 (UTM)

현재: CRM에서 UTM 파라미터를 미리보기로 보여주기만 함.

**안 되는 것**:
- 발송 시 이메일/알림톡 본문 링크에 UTM 자동 주입
- GA4/BigQuery에서 UTM 기반 클릭 유입 집계

**현재 UTM 규칙** (CRM에서 생성하는 형태):
```
utm_source = email | alimtalk
utm_medium = crm
utm_campaign = {sku}_{slug}
utm_term = {context}
utm_content = {campaign_slug}-cta{N}
```

### 유입률 계산

현재: mock 하드코딩.

**계산하려면 필요한 행위**:
1. 발송 로그에서 당일 수신 회사 목록 추출
2. 접속 로그에서 같은 날 접속한 회사 목록 추출
3. 교집합 계산
4. 발송 다음 날 아침에 배치 실행 (전일 접속 로그 확정 후)

### 알림톡 발송 결과

현재: NHN 응답의 resultCode만 확인, 적재 안 함.

**안 되는 것**:
- 건별 성공/실패 기록
- 알림톡 실패 시 SMS 대체 발송 처리

## 3. 자동 발송

### 시간 기반

CRM에 "자동화 현황"으로 표시만 됨. 실제 발송은 안 일어남.

| 자동화 | 스케줄 | 현재 상태 |
|--------|--------|-----------|
| 자금일보 | 매일 09:00 | 표시만 |
| 월요일 주간 요약 | 월요일 08:00 | 표시만 |
| 세무대리인 월간 리포트 | 매월 1일 | 표시만 |

**필요한 행위**: 크론 스케줄러가 시간 도래 시 대상 회사 쿼리 → 발송 실행

### 조건 기반

| 자동화 | 트리거 조건 | 현재 상태 |
|--------|------------|-----------|
| 가입 D+1 연동 넛징 | 회원가입 후 24시간, 데이터 미연동 | 표시만 |
| 미접속 수집 중단 안내 | 7/15/30일 연속 미접속 | 표시만 |
| 수집 오류 알림 | 스크래핑 FATAL 에러 | 표시만 |

**필요한 행위**:
- 이벤트 소스 연결 (회원가입 웹훅, 접속 로그 배치, 에러 콜백)
- 이벤트 수신 → 조건 평가 → 발송 실행

### 발송 정책 적용

CRM 설정 페이지에서 값 설정은 가능. 실제 적용은 안 됨.

| 정책 | UI 상태 | 실제 적용 |
|------|---------|-----------|
| Daily Cap (하루 3통) | 숫자 조절 가능 | 발송 시점에 체크 안 함 |
| 주말 발송 금지 | 토글 ON/OFF 가능 | 주말에도 발송됨 |
| 공휴일 발송 금지 | 토글 + 공휴일 목록 관리 가능 | 공휴일에도 발송됨 |

**필요한 행위**: 발송 실행 직전에 cap/주말/공휴일 체크 → 위반 시 다음 영업일로 지연

## 4. 모니터링

### 워닝 자동 감지

CRM 워닝 페이지에 3가지 섹션이 있지만 전부 mock.

| 워닝 유형 | 감지 조건 | 현재 상태 |
|-----------|-----------|-----------|
| CTR 미달 | 7일 CTR < 기준값 | mock 8건 |
| 추적 미수집 | 발송 성공인데 오픈/클릭 데이터 없음 | mock 3건 |
| 휴면 템플릿 | 7일 일평균 발송 < 3건 | mock 4건 |

**필요한 행위**: 실 발송 로그 기반으로 매일 집계 → 임계치 초과 항목 자동 추출

### 실패 알림

| 알림 조건 | 현재 상태 |
|-----------|-----------|
| 발송 실패율 5% 초과 | 차트에 점선만 |
| NHN API 연속 실패 | 감지 안 함 |
| 수신거부율 0.5% 초과 | 차트에 점선만 |

**필요한 행위**: 임계치 초과 시 슬랙 `#crm-alerts` 채널에 자동 알림

### 사이드바 워닝 뱃지

현재: mock 데이터 합산 (페이지 로드 시 1회 계산).
필요: 실데이터 기반 + 주기적 갱신 (폴링 or 실시간 구독)

## 5. 연동 흐름 요약

```
CRM "발송 요청" 클릭
  ↓
[정책 체크] cap / 주말 / 공휴일
  ↓ 통과                ↓ 위반
  ↓                   지연 (다음 영업일)
  ↓
[NHN API 호출] 이메일 or 알림톡
  ↓
[결과 수집] 성공/실패/오픈
  ↓
[지표 계산] 오픈율, 유입률 (다음 날 배치)
  ↓
[모니터링] 임계치 초과 → 슬랙 알림
  ↓
[CRM 표시] 대시보드 차트, 발송이력, 워닝
```

## 6. 우선순위

| 순서 | 액션 | 없으면 |
|------|------|--------|
| **1** | 이메일 실발송 | "발송 요청" 눌러도 안 나감 |
| **2** | 알림톡 대량 발송 | CSV 업로드했는데 1건만 보냄 |
| **3** | 발송 결과 수집 | 오픈율·유입률 영원히 null |
| **4** | 오픈 추적 | 이메일 오픈율 계산 불가 |
| **5** | 유입률 배치 | 핵심 지표 계산 불가 |
| **6** | 시간 기반 스케줄러 | 자금일보 등 매일 수동 발송해야 함 |
| **7** | 조건 기반 트리거 | 온보딩/리텐션 자동화 안 됨 |
| **8** | 정책 적용 | cap 설정해도 무제한 발송됨 |
| **9** | 슬랙 알림 | 장애 시 아무도 모름 |
